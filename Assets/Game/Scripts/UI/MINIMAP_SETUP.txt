   МИНИ-КАРТА НА СТАТИЧНОЙ ТЕКСТУРЕ (без дополнительной камеры)
═══════════════════════════════════════════════════════════════════════════════

1. ПОДГОТОВКА ТЕКСТУРЫ КАРТЫ
   • Скриншот уровня сверху или нарисованная схема. Импортируйте в Unity.

2. UI В CANVAS
   • Панель "Minimap" → внутри Raw Image с текстурой карты.
   • Иконка игрока (Image) — дочерняя к этому Raw Image (скрипт сам перенесёт, если нужно).

3. ИГРОВАЯ ОБЛАСТЬ (важно — иначе точка в углу)
   Мини-карта мапит мир в прямоугольник. Нужно задать, какой участок мира (XZ)
   соответствует текстуре карты.

   Вариант A — маркеры в сцене (рекомендуется):
   • Создайте два пустых объекта (Create Empty): "MapCornerMin" и "MapCornerMax".
   • Поставьте один в левый нижний угол игровой зоны по XZ, второй — в правый верхний.
   • В Minimap UI назначьте: World Corner Min = MapCornerMin, World Corner Max = MapCornerMax.
   • Границы возьмутся из позиций этих объектов — область задаётся наглядно.

   Вариант B — Terrain:
   • Включите "Use Terrain Bounds If No Markers". Если в сцене есть Terrain и маркеры
     не заданы, границы возьмутся из размера террейна.

   Вариант C — числа вручную:
   • Оставьте маркеры пустыми и выключите Use Terrain Bounds.
   • Заполните World Min Fallback и World Max Fallback (X и Z левого нижнего и правого
     верхнего угла игровой области в мировых координатах).

4. ОСТАЛЬНОЕ
   • Map Graphic: тот Raw Image / Image, на котором отображается карта.
   • Player Icon: иконка игрока (будет дочерней к карте).
   • При необходимости включите Log Mapping Once — в консоль выведется маппинг
     мира в норм. координаты для отладки.

Итог: игровая область задаётся маркерами или Terrain/числами; точка игрока
корректно переводится в позицию на мини-карте.
