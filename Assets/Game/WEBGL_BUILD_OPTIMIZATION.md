# Оптимизация сборки WebGL

Сборка WebGL использует IL2CPP (C# → C++ → компиляция), поэтому 10+ часов — не редкость для больших проектов. Ниже — что уже сделано и что можно сделать дополнительно.

---

## Уже применено в проекте

1. **Photon Demos исключены из WebGL**  
   В `PhotonUnityNetworking.Demos.asmdef` задано `excludePlatforms: ["WebGL"]`. Демо PUN (PunCockpit, DemoSlotRacer, PunBasics и т.д.) не компилируются и не попадают в билд WebGL — меньше кода и быстрее сборка.

2. **Исключения: Full Without Stacktrace**  
   В Player Settings → WebGL → Publishing Settings: **Exception support** = **Full Without Stacktrace** (`webGLExceptionSupport: 2`). Поддержка исключений есть, но без стека — сборка быстрее, чем при Full With Stacktrace.

3. **IL2CPP Code Generation: Optimize Size**  
   Для платформы WebGL включена генерация кода с упором на размер/время сборки (OptimizeSize), а не на максимальную скорость рантайма. Ускоряет компиляцию и уменьшает размер билда.

## Рекомендации (сделать вручную)

### Сборка и папки

- **Собирать в одну и ту же папку**  
  Не удалять папку билда между сборками. IL2CPP пересобирает только изменённые файлы — инкрементальная сборка может сократить время с часов до десятков минут при правках кода.

- **Папки проекта и билда на SSD**  
  Проект и выходная папка WebGL должны лежать на SSD. Сильно ускоряет чтение/запись при компиляции.

- **Отключить антивирус для папок проекта и билда**  
  Исключить из сканирования в реальном времени:
  - папку проекта (например, `D:\PROJECTS\Test\Test`);
  - папку сборки WebGL (например, `Build\WebGL`).  
  В тестах Unity отключение Windows Defender давало ускорение сборки на 50–66%.

### Уменьшение объёма кода

- **Убрать неиспользуемые плагины из билда**  
  Если для WebGL не нужны:
  - демо и примеры (TextMesh Pro Examples, Unity Effect Examples и т.п.),
  - целые плагины (например, Farland Skies, лишние демо Photon),  
  их можно не включать в сцену/префабы и по возможности вынести в отдельные asmdef с `excludePlatforms: ["WebGL"]`, чтобы они не попадали в билд.

- **Strip Engine Code**  
  Уже включено (`stripEngineCode: 1`) — неиспользуемый код движка отсекается.

### Дополнительные настройки (по необходимости)

- **Exception support: None**  
  Если в игре не нужны try/catch в билде WebGL, в Publishing Settings можно поставить **None**. Максимальное ускорение сборки и уменьшение размера, но без поддержки исключений.

- **Managed Stripping Level**  
  Сейчас для WebGL стоит уровень 1 (Minimal). Для ещё меньшего размера можно попробовать **Low** или **Medium** (2–3). Проверить игру после сборки: рефлексия и динамическая загрузка могут сломаться.

- **Анализ размера билда**  
  В Player Settings → WebGL включить **Analyze build size** и посмотреть, какие ассеты и скрипты больше всего весят. Это поможет решить, что выносить в asmdef или отключать только для WebGL.

## Почему WebGL собирается так долго

- **IL2CPP** переводит весь C# в C++ и затем компилирует его (как большой нативный проект).
- **Emscripten** компилирует этот C++ в WebAssembly — ещё один тяжёлый этап.
- **Объём кода**: 367+ C# файлов (включая Photon, плагины, примеры) дают много кода для конвертации и компиляции.

Исключение демо, настройки исключений и Optimize Size уже уменьшают время и размер; инкрементальная сборка в одну папку и SSD/антивирус дают самый большой выигрыш при повторных сборках.
